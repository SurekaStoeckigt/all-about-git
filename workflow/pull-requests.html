Git Workflow

Pull Requests

Why use pull requests?

Pull requests allow you to request that the owner of an original (sometimes 

upstream
) repository pull changes you have made into the main repo.

Opening a pull request allows you and the code owner to discuss the changes you have made in a threaded discussion on Github.

Pull requests are different to using 

git pull
, which will 
fetch
and 
merge
changes from a remote into your local repository.

How do you use pull requests?

Once you have made changes, stage and commit them on a new branch (see Branching). Then push the changes up to your remote repository (usually 

origin
) and click the 'Create Pull Request' button on Github. You can then choose which repository you are requesting the pull request for, and which codebase you want to pull in.

Once you have submitted the pull request, communicate any difficulties with the code owner in the threaded discussion. They might ask you to change some code, especially if it is raising conflicts for them. Once they are satisfied with the changes you have made, the code owner can confirm the pull request from the thread. You've just contributed your code to the codebase!
